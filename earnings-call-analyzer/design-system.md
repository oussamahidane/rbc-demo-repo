# Report Design System

> **Why this file exists.** Every report renders identically because its structure,
> tone, and formatting live here — in one markdown file — not scattered through code
> or prompts. This is the `.md`-as-context discipline made tangible: the same practice
> that fixed the $5-token problem now guarantees a consistent house style. Change the
> house style by editing this file; the renderer reads it.

The renderer (`eca/report.py`) parses the `config` block below. Prose outside it is
guidance for humans writing or reviewing reports.

```config
title_format: {company} {quarter} Earnings Call Analysis
disclaimer: Generated by the Earnings-Call Analyzer for demonstration. Hypotheses offered for correction, not investment advice. In production this runs on approved infrastructure with data governance and model-risk review. The personal API key is demo-only.
citation_note: Every figure and material claim cites its source line as [Ln]. Uncited claims are flagged, not hidden.
sections: executive_summary, guidance, key_metrics, management_tone, risks, qa_highlights, notable_quotes
tone: institutional, measured, hypotheses-not-verdicts
```

## Typography & structure rules

- **One H1**: the report title, from `title_format`.
- **A metadata line** directly under the title: source file, model used, run timestamp, grounding coverage.
- **Section order is fixed** by `sections` above. Skip a section only if it has no content; never reorder.
- Section headings are H2, in Title Case, human-readable (`Executive Summary`, not `executive_summary`).
- **Key Metrics** renders as a table: Metric | Value | Citation.
- Every other section is bulleted; each bullet ends with its `[Ln]` citation(s).
- **Notable Quotes** are blockquotes, each attributed and cited.

## Tone rules

- Measured and institutional. State magnitudes, not adjectives ("revenue +14% YoY [L42]", not "revenue soared").
- Frame reads as hypotheses for correction where judgment is involved — never verdicts.
- Never invent a figure. If the transcript doesn't support a claim, omit it. A missing
  number is acceptable; a hallucinated one is disqualifying.
- Sentiment is reported as an established-baseline signal (see `docs/methodology.md`),
  not as a novel discovery.
